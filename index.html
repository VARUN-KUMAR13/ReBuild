About the Event

What is
Website Rebuilding?

01

The Challenge

Participants receive an existing popular website or webpage as a reference. Your mission: completely rebuild it with better UI, improved structure, and enhanced functionality that surpasses the original.

02

Your Freedom

You are not restricted to copying the design. Think critically about UX flaws in the original, reimagine the layout, improve accessibility, and modernize the visual language entirely.

03

Technologies

HTML, CSS, JavaScript, React, Vue, Angular, Next.js, Tailwind — use any modern web technology stack you're comfortable with. The tools don't matter; the output does.

04

The Outcome

Walk away with a portfolio-worthy project, hands-on experience with real-world redesign challenges, peer recognition, and the satisfaction of shipping something genuinely better.

Technical Rules

The Rules of
the Game

WHAT YOU MUST NOT DO

  • ✗ Do not use backend, servers, or databases
  • ✗ Do not use external API calls
  • ✗ Do not remove existing sections of the page
  • ✗ Do not add backend form submission

WHAT TO DO

  • ✓ Follow the given instructions and file structure
  • ✓ Keep the design clean, readable, and well aligned
  • ✓ Ensure the page runs by simply opening the HTML file
  • ✓ Make the page responsive for mobile devices
  • ✓ Work on the Login/Sign Up pages and ensure Navbar items function properly
  • ✓ Add necessary functionalities in the website and improve the overall UI/UX experience
